# Returns Assistant

Returns Assistant is a Django REST Framework API for handling ecommerce return requests. When a customer submits a return, the API sends the product and return reason to Google Gemini, stores the AI decision, and returns the completed record as JSON.

The AI classifies each request as:

- `APPROVE` - clearly valid returns such as damaged, defective, wrong item, or never arrived.
- `EXCHANGE` - issues that can likely be solved with a replacement, size change, or similar swap.
- `ESCALATE` - vague, suspicious, policy-sensitive, or AI-unavailable cases that need manual review.

## Tech Stack

- Python
- Django
- Django REST Framework
- SQLite
- Google Gemini API via `google-generativeai`
- `python-dotenv` for local environment variables

## How It Works

1. A client sends a return request to `POST /api/returns/`.
2. `ReturnRequestSerializer` validates the customer-provided fields.
3. `get_return_decision()` in `returns/ai_engine.py` sends the product name and return reason to Gemini.
4. The response is parsed into an `ai_decision` and `ai_explanation`.
5. The request is saved in SQLite through the `ReturnRequest` model.
6. The API returns the stored return request, including the AI-generated fields.

If Gemini fails or returns an unexpected result, the app falls back to `ESCALATE` with a manual-review explanation instead of crashing the API request.

## Setup

### 1. Create and activate a virtual environment

```bash
python -m venv venv
```

On Windows PowerShell:

```powershell
.\venv\Scripts\Activate.ps1
```

On macOS/Linux:

```bash
source venv/bin/activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure environment variables

Create a `.env` file in the project root:

```env
GEMINI_API_KEY=your_gemini_api_key_here
```

You can create a Gemini API key from Google AI Studio: https://aistudio.google.com/app/apikey

### 4. Run database migrations

```bash
python manage.py migrate
```

### 5. Start the development server

```bash
python manage.py runserver
```

The API will be available at:

```text
http://127.0.0.1:8000/
```

## API Reference

### Create a Return Request

```http
POST /api/returns/
```

Request body:

```json
{
  "order_id": "ORD-1042",
  "product_name": "Running Shoes Size 10",
  "reason": "Received the wrong size. I ordered size 9."
}
```

Example response:

```json
{
  "id": 1,
  "order_id": "ORD-1042",
  "product_name": "Running Shoes Size 10",
  "reason": "Received the wrong size. I ordered size 9.",
  "ai_decision": "EXCHANGE",
  "ai_explanation": "The issue can be resolved by replacing the item with the correct size.",
  "created_at": "2026-04-27T10:30:00Z"
}
```

Only `order_id`, `product_name`, and `reason` are accepted from the client. The `id`, `ai_decision`, `ai_explanation`, and `created_at` fields are generated by the server.

### List Return Requests

```http
GET /api/returns/
```

Returns all stored return requests, newest first.

### Get One Return Request

```http
GET /api/returns/<id>/
```

Returns a single return request by ID. If the ID does not exist, the API returns a `404` response:

```json
{
  "error": "Return request with id 123 not found."
}
```

## Example cURL Requests

Create a return request:

```bash
curl -X POST http://127.0.0.1:8000/api/returns/ \
  -H "Content-Type: application/json" \
  -d '{"order_id":"ORD-1042","product_name":"Running Shoes Size 10","reason":"Received the wrong size. I ordered size 9."}'
```

List return requests:

```bash
curl http://127.0.0.1:8000/api/returns/
```

Fetch one return request:

```bash
curl http://127.0.0.1:8000/api/returns/1/
```

## Admin

The `ReturnRequest` model is registered in Django admin with search, filtering by AI decision, and read-only AI fields.

Create an admin user:

```bash
python manage.py createsuperuser
```

Then visit:

```text
http://127.0.0.1:8000/admin/
```

## Development Notes

- The app uses SQLite by default through `db.sqlite3`.
- `ALLOWED_HOSTS` is set to `['*']` and `DEBUG` is enabled, so the settings are intended for local development.
- API responses are available as JSON and through DRF's browsable API.
- The Gemini model is configured in `returns/ai_engine.py`.
